# Project Context

## 背景

Diet Cloud 起源于一个个人减脂记录需求：用户每天拍餐食照片，希望快速记录早餐、午餐、晚餐和其他进食，并自动估算热量、重量和营养成分。随后项目扩展为一个综合健康复盘工具，加入体重体脂、身体分布、Apple 健康运动数据、趋势图和目标系统。

项目目标不是做一个通用的营销网站，而是一个长期每天使用的私人健康记录面板。核心体验是：

- 手机端可以快速查看、添加和复盘。
- 电脑端可以更完整地分析趋势和明细。
- 数据和照片保存在云端，避免依赖本地文件。
- AI 只作为估算和录入辅助，用户始终可以手动修正。

## 设计思路

UI 方向参考 Apple Health、Apple Fitness、iOS/macOS 的系统视觉语言：

- 内容优先，减少装饰性元素。
- 使用清晰分层、柔和阴影、半透明卡片和低饱和色。
- 深色模式要求高对比，不牺牲可读性。
- 移动端和桌面端采用不同密度：手机更像原生 App，桌面更像仪表盘。

## 产品模块

1. 饮食记录
   - 餐次：早餐、午餐、晚餐、其他。
   - 支持照片、手动录入、AI 分析、常吃食物一键添加。
   - 统计热量、蛋白质、碳水、脂肪、膳食纤维。

2. 身体数据
   - 记录体重、BMI、体脂率、肌肉量、水分、基础代谢等。
   - 支持体脂秤截图识别。
   - 展示身体分布和指标历史趋势。

3. 运动数据
   - 支持 Apple 健康导出文件导入。
   - 支持 iPhone Shortcuts 调用接口同步每日数据。
   - 展示步数、活动能量、运动时间、距离、站立时间、心率和睡眠等指标。

4. 趋势分析
   - 周期热量、蛋白质、纤维、体重和活动量趋势。
   - 减脂复盘、替换建议、明日建议。

5. 照片库
   - 按日期浏览餐食照片。
   - 图片可放大查看，并关联餐次和热量。

## 当前架构选择

- 前端：Vite + React，部署为静态资源。
- 数据：Supabase Postgres，RLS 按用户隔离。
- 图片：Supabase Storage。
- 服务端：Vercel Functions 处理 AI 分析、service role 写入和 Shortcuts 同步。
- AI：Gemini 多模型 fallback，用于图片估算。

## 后续开发原则

- 先保证数据准确和可编辑，再优化视觉。
- 所有自动估算都应该保留用户确认和修正入口。
- 新增功能要同时考虑手机端和桌面端。
- 不要把 service role key 或任何真实 token 放进前端代码。
